# Breaking changes

This file enumerates the customer-visible changes the facade could not
absorb silently. Each entry is generated by `spec-idiomatic` during a
regen of `pkg/` against a new `api/` snapshot.

## TranscribeInput.MinDuration removed (spec PR #8, @internal audit)

- **Tier:** 2 (unavoidable-breaking)
- **Shape:** `spectypes.TranscribeInput` (generated, in `api/types`)
- **Member:** `MinDuration *string`
- **Rationale:** Spec PR #8 (`feat(traits): add @internal + tag
  undocumented stem fields`) removed this query parameter from the
  public spec as part of an audit of stem-internal parameters that
  were never publicly documented. Customers setting this parameter
  were reaching into an undocumented stem surface.
- **Customer-visible impact:** The facade no longer exposes
  `PreRecordedTranscriptionOptions.MinDuration`. Customer code that
  set this field will fail to compile on this SDK upgrade.
- **Facade behaviour:** Field removed from the facade options struct;
  wiring block removed from `optionsToTranscribeInput`; matching
  `TestWires_MinDuration` removed (the field no longer exists on the
  facade, so neither `TestWires_*` nor `TestDropped_*` applies).
- **Migration required:** Remove references to `MinDuration` from
  callers. There is no replacement on the public API surface.
