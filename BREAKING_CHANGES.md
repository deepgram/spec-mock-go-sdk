# Breaking Changes

This file is the cumulative record for tier 1 and tier 2 regen changes.
See `REGEN.md` section 15 for the full absorption rules used during codegen review.

Tier 0 changes are silently absorbed: new wire fields are added, wired,
and covered by tests without an entry here.

Tier 1 changes are absorbed with ceremony: removed wire fields stay public
but deprecated; converter wiring is removed and entries go here.

Tier 2 changes are unavoidable breaks: removed wire and public options fields
both disappear, and entries go here with review label context.

## No tier 1 or tier 2 changes have landed yet.

## 2026-05-20 — Transport-pluggable facade

`pkg/client/listen/v1/rest` and `pkg/client/listen/v1/websocket` are
removed and replaced by per-operation packages with transport-binding
constructor options (per ADR-0004 update in deepgram/spec):

- `pkg/client/listen/v1/prerecorded` (package `prerecordedv1`) replaces
  the old `rest` package. Constructor: `New(opts ...Option)`. Options
  include `WithHTTPTransport()` (default) and
  `WithSageMakerTransport(client, endpointName)`.
- `pkg/client/listen/v1/live` (package `livev1`) replaces the old
  `websocket` package. Constructor: `New(opts ...Option)`. Options
  include `WithWebSocketTransport()` (default),
  `WithSageMakerBidiTransport(...)`, and `WithWebRTCTransport(...)`.

`PreRecordedTranscriptionOptions` and `LiveTranscriptionOptions` keep
their names and field semantics. `AdditionalQueryParams` is now
transport-aware: HTTP/WebSocket map to URL query string, SageMaker maps
to `X-Amzn-SageMaker-Custom-Attributes` header via `url.Values.Encode()`.

Migration: rename imports, swap `rest.NewWithDefaults()` for
`prerecorded.New(prerecorded.WithCredentials(apiKey, accessToken))`
(or pass `WithAPIKey(...)` / `WithAccessToken(...)`). Same for
`ws.NewWithDefaults()` -> `live.New(live.WithCredentials(...))`.

The hand-written tests under the old packages and the example programs
under `examples/` are not yet migrated. Examples are gated behind the
build tag `examples_migration_pending` so they do not break the build;
they will be regenerated by spec-idiomatic alongside the new packages'
test coverage in a follow-up.

### Note on examples/

The hand-written example programs that lived under `examples/` are
deleted in this change. They referenced `rest.NewWithDefaults()` and
`ws.NewWithDefaults()`, which no longer exist on the new packages.
spec-idiomatic will author replacement examples for the new
`prerecorded` and `live` packages in the step-4 facade-test
regeneration pass.
