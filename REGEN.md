# REGEN — Spec pipeline + maintainer guide

This document is for **maintainers** of the SDK. If you just want to use
it, [`README.md`](README.md) is the right place.

## What this repo is

The Go consumer of the Deepgram Smithy spec pipeline. `api/` holds
machine-generated wire types from `deepgram/spec-codegen-go`. `pkg/`
holds the idiomatic Go facade that absorbs spec wobbles so customer
call signatures stay stable. The `deepgram/spec-idiomatic` agentic
runner maintains the facade on every spec change. It's the prototype
of the spec-driven replacement for `deepgram/deepgram-go-sdk`.

Listen is the only product wired today (REST + WebSocket). Other
products return per-product as each migrates into the pipeline.

## The four-repo pipeline

```
deepgram/spec               (Smithy IDL, source of truth)
       │
       ▼
deepgram/spec-codegen-go    (Java SmithyBuildPlugin)
       │
       │  emits api/
       ▼
deepgram/spec-mock-go-sdk   (this repo — api/ + pkg/)
       ▲
       │  regenerates pkg/ on every api/ PR
       │
deepgram/spec-idiomatic     (agentic facade regenerator)
```

When a spec change merges in `deepgram/spec`, the fan-out workflow
there opens a PR here with `api/` regenerated. `spec-idiomatic` fires
on that PR, regenerates `pkg/`, runs `go test ./...`, and pushes any
edits back to the PR branch. A human reviewer acks and merges.

## Layout

| Path | Edit policy |
|---|---|
| `api/` | Machine-generated. Every file begins with `// Code generated by smithy-go-codegen DO NOT EDIT.`. Touching anything here will be wiped on the next regen. |
| `pkg/` | Idiomatic Go facade. Edit freely. The facade exists to absorb api/ wobbles so customer signatures stay stable. |
| `.agents/skills/` | Maintainer-facing skills loaded by `spec-idiomatic` at regen time. |
| `AGENTS.md` | Repo orientation for maintainers, skill index, substrate philosophy. |
| `llms.txt` | Hand-curated retrieval-engine entry point per [llmstxt.org](https://llmstxt.org/). |

## Conventions for `pkg/`

Three top rules:

1. **Don't edit `api/`.** It's regenerated.
2. **Public surface ships with `Example_*` tests.** Every exported
   type or function in `pkg/` has at least one `Example_*` in an
   adjacent `*_test.go`. `go test ./...` runs them as smoke tests.
3. **One concept per file.** No grab-bag `utils.go` or files that
   co-locate REST and WebSocket types.

## How a change lands

1. Someone merges a change in `deepgram/spec` (the Smithy IDL).
2. The fan-out workflow opens a PR in this repo with `api/` regenerated.
3. `spec-idiomatic` fires on that PR, regenerates the `pkg/` facade,
   runs `go build ./...`, posts a work-done checklist comment, and
   pushes any edits back to the PR branch.
4. Human reviewer checks the diff per
   [`sdk-pr-review`](.agents/skills/sdk-pr-review/SKILL.md), acks any
   breaking ceremony documented in
   [`spec-idiomatic/prompts/system.md`](https://github.com/deepgram/spec-idiomatic/blob/main/prompts/system.md),
   merges.

## Build and test

```bash
go build ./...
go test ./... -count=1
```

Both must pass for an auto-regen PR to land. `go test` runs every
`Example_*` function in the package; if a runnable example stops
compiling, the build fails.

## Cross-repo context

This repo is one of four:

- [`deepgram/spec`](https://github.com/deepgram/spec) — Smithy IDL source of truth.
- [`deepgram/spec-codegen-go`](https://github.com/deepgram/spec-codegen-go) — Java SmithyBuildPlugin that emits `api/`.
- [`deepgram/spec-mock-go-sdk`](https://github.com/deepgram/spec-mock-go-sdk) — this repo.
- [`deepgram/spec-idiomatic`](https://github.com/deepgram/spec-idiomatic) — agentic facade regenerator.

## Substrate philosophy

The **source code is the substrate**. `Example_*` functions, the
README opening paragraph, and `llms.txt` are **derived** from it. When
the substrate changes, the derived layer must be regenerated to
match — that's `spec-idiomatic`'s job, not yours, but you are
responsible for verifying it in review.

Prose that contradicts the substrate is wrong by definition. If you
find a mismatch, fix the prose, not the code.
