# spec-mock-go-sdk

spec-mock-go-sdk is the prototype Go SDK generated by the Deepgram
Smithy spec pipeline. It is the in-development replacement for
`deepgram/deepgram-go-sdk`: `api/` holds the machine-generated wire
types from `deepgram/spec-codegen-go`, `pkg/` holds the idiomatic Go
facade that absorbs spec wobbles so customer call signatures stay
stable across regenerations, and the `deepgram/spec-idiomatic`
agentic runner maintains the facade on every spec change. Runs on
Go 1.22+. Prototype: only `/v1/listen` is wired through the new
pipeline today; the rest of the surface is legacy carried over from
`deepgram/deepgram-go-sdk` and gets cut over product by product.

[![Go Reference](https://pkg.go.dev/badge/github.com/deepgram/spec-mock-go-sdk.svg)](https://pkg.go.dev/github.com/deepgram/spec-mock-go-sdk)

## What this repo is for

This is the in-development Go SDK output of the spec pipeline. It
sits at the bottom of a four-repo flow:

```
deepgram/spec               (Smithy IDL, source of truth)
       │
       ▼
deepgram/spec-codegen-go    (Java SmithyBuildPlugin)
       │
       │  emits api/
       ▼
deepgram/spec-mock-go-sdk   (this repo — api/ + pkg/ + tests + examples)
       ▲
       │  regenerates pkg/ on every api/ PR
       │
deepgram/spec-idiomatic     (agentic facade regenerator)
```

When a spec change merges in `deepgram/spec`, the fan-out workflow
there opens a PR here with `api/` regenerated. `spec-idiomatic` fires
on that PR, regenerates `pkg/`, runs `go test ./...`, and pushes any
edits back to the PR branch. A human reviewer acks and merges.

## Layout

| Path | Edit policy |
|---|---|
| `api/` | Machine-generated. Every file begins with `// Code generated by smithy-go-codegen DO NOT EDIT.`. Touching anything here will be wiped on the next regen. |
| `pkg/` | Idiomatic Go facade. Edit freely. The facade exists to absorb api/ wobbles so customer signatures stay stable. |
| `examples/` | Runnable code samples per transport / use case. |
| `tests/` | Unit and integration tests. |
| `.agents/skills/` | Maintainer-facing skills loaded by `spec-idiomatic` at regen time. |
| `AGENTS.md` | Repo orientation, skill index, substrate philosophy. |
| `llms.txt` | Hand-curated retrieval-engine entry point per [llmstxt.org](https://llmstxt.org/). |

## Conventions

The full conventions are in
[`AGENTS.md`](AGENTS.md) and the six skills under
[`.agents/skills/`](.agents/skills/). Three top rules:

1. **Don't edit `api/`.** It's regenerated.
2. **Public surface ships with `Example_*` tests.** Every exported
   type or function in `pkg/` has at least one `Example_*` in an
   adjacent `*_test.go`. `go test ./...` runs them as smoke tests.
3. **One concept per file.** No grab-bag `utils.go` or files that
   co-locate REST and WebSocket types.

See [`sdk-agentic-readiness`](.agents/skills/sdk-agentic-readiness/SKILL.md)
for why and the full ruleset.

## Build and test

```bash
go build ./...
go test ./... -count=1
```

Both must pass for an auto-regen PR to land.

## Auth (in real use)

The plumbed surfaces use the existing env-driven auth from the broader
Deepgram Go SDK pattern:

```bash
export DEEPGRAM_API_KEY="your_api_key"
# or
export DEEPGRAM_ACCESS_TOKEN="your_access_token"
```

`rest.NewWithDefaults()` picks them up at construction.
