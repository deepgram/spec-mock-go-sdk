# Local regen rehearsal

`spec-idiomatic` runs in CI on every api/ PR, but you can run it on
your laptop against a synthetic change. Useful for:

- Testing a hypothetical spec change before pushing it to
  `deepgram/spec`.
- Reproducing a CI regen failure locally to iterate faster.
- Validating that the facade absorbs a new shape correctly.
- Stress-testing prompt or classifier changes in `spec-idiomatic`.

## Prerequisites

- `spec-idiomatic` cloned next to `spec-mock-go-sdk` at
  `~/Projects/deepgram/spec-idiomatic` (or wherever your sibling
  checkouts live).
- JDK 21 on PATH (`/opt/homebrew/opt/openjdk@21/bin` on macOS).
- Go on PATH.
- `ANTHROPIC_API_KEY` in your environment for the `--apply` step.
  (Without it, the orchestrator runs in dry-run mode and prints the
  prompt instead of calling the LLM.)
- The orchestrator built once:
  ```bash
  cd ~/Projects/deepgram/spec-idiomatic
  gradle --quiet -Dorg.gradle.java.installations.paths=$JAVA_HOME build -x test
  (cd parsers/go && go build -o parser .)
  ```

## The flow

### 1. Snapshot api/ as "before"

The orchestrator diffs a `before/api/` snapshot against the current
`api/` to build its change plan. The simplest snapshot is a tree
copy:

```bash
cd ~/Projects/deepgram/spec-mock-go-sdk
rm -rf /tmp/dx-local-run && mkdir -p /tmp/dx-local-run/before
cp -r api /tmp/dx-local-run/before/api
```

### 2. Mutate api/ as if a spec regen had landed

Edit `api/types/types.go` (or wherever else) the way a real regen
would produce. Hand-edit for plausibility — preserve the
`// Code generated by smithy-go-codegen DO NOT EDIT.` header, keep the
Smithy-style doc comments on each new field. Examples of plausible
mutations:

- Add a new optional field to an existing struct.
- Add a new union variant (struct + `ServerStreamMember*` wrapper +
  `isServerStream()` method).
- Remove an existing field (tests the absorbed-breaking path).
- Widen a field type (`*float32` → `*float64`).
- Add a new struct entirely (TYPE_ADDED).

### 3. Confirm the baseline

```bash
go build ./...
```

If the mutation is additive (e.g. a new field that nothing references
yet), this should pass. If the mutation is breaking (e.g. removed a
field the facade depends on), this will fail — that's expected, the
orchestrator's job is to make it pass again.

### 4. Run spec-idiomatic with --apply

```bash
export JAVA_HOME=/opt/homebrew/opt/openjdk@21
export PATH=$JAVA_HOME/bin:$PATH
cd ~/Projects/deepgram/spec-idiomatic
export SPEC_IDIOMATIC_GO_PARSER=$PWD/parsers/go/parser

gradle --quiet -Dorg.gradle.java.installations.paths=$JAVA_HOME run --args="\
  --before-api /tmp/dx-local-run/before/api \
  --after-api  ~/Projects/deepgram/spec-mock-go-sdk/api \
  --facade-dir ~/Projects/deepgram/spec-mock-go-sdk/pkg \
  --language   go \
  --apply \
  --max-retries 3"
```

Expected output on success:

```
[spec-idiomatic] language=go apply=true maxRetries=3
[spec-idiomatic] plan: N additive, M breaking
[spec-idiomatic] ast: 151 files
[spec-idiomatic] attempt 1/3
[spec-idiomatic] applied K edits, L breaking notes
[spec-idiomatic] build passed on attempt 1
```

If breaking notes are non-zero, the runner also prints the structured
summary and points at `BREAKING_CHANGES.md` at the repo root.

### 5. Inspect the diff

```bash
cd ~/Projects/deepgram/spec-mock-go-sdk
git status --short
git diff pkg/
cat BREAKING_CHANGES.md 2>/dev/null
```

Verify:

- Customer-facing types in `pkg/` did not lose fields (unless the
  ceremony asked you to).
- Converters use the existing deref helpers, no ad-hoc nil checks.
- `Example_*` tests (when present) still compile and pass under
  `go test ./...`.
- `BREAKING_CHANGES.md` content is accurate and one-concept-per-entry.

### 6. Restore

```bash
cd ~/Projects/deepgram/spec-mock-go-sdk
git checkout -- api/ pkg/
rm -f BREAKING_CHANGES.md
rm -rf /tmp/dx-local-run
go build ./...   # confirm restored
```

## Dry-run (no LLM call)

If you want to inspect the assembled prompt without burning Anthropic
tokens, omit `--apply`:

```bash
gradle --quiet -Dorg.gradle.java.installations.paths=$JAVA_HOME run --args="\
  --before-api /tmp/dx-local-run/before/api \
  --after-api  ~/Projects/deepgram/spec-mock-go-sdk/api \
  --facade-dir ~/Projects/deepgram/spec-mock-go-sdk/pkg \
  --language   go"
```

The full prompt prints to stdout. Useful for prompt-engineering work in
`spec-idiomatic/prompts/`.

## Gotchas

1. The orchestrator's `BuildVerifier` walks up from `--facade-dir` to
   find `go.mod`. If you point it at a directory without a `go.mod`
   ancestor, the build step fails immediately. Use absolute paths.
2. The Go AST parser is invoked once per run. Prebuild the binary
   (`go build -o parser .` in `parsers/go/`) and set
   `SPEC_IDIOMATIC_GO_PARSER` to avoid `go run` overhead on every
   invocation.
3. Don't commit `BREAKING_CHANGES.md` from a local rehearsal — it's a
   real artifact intended for real PRs. The `--apply` runs on your
   laptop will write one; remove it before pushing.
4. The orchestrator never edits `api/`. If your mutation introduces
   inconsistency in `api/` (e.g. you added a `ServerStreamMember*`
   without updating `api/transport/websocket/streaming.go`), fix that
   by hand or by running smithy regen properly — the facade can't
   compile against a half-mutated wire layer.
